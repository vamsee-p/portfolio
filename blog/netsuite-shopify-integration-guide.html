Why Integrate NetSuite with Shopify?

Growing e-commerce businesses using both Shopify and NetSuite quickly hit a painful wall: the two systems don't talk to each other. Orders flow into Shopify but someone has to manually copy them into NetSuite. Inventory updates in NetSuite don't reflect on the Shopify storefront. Refunds require updates in both systems separately.

The result? Hours of manual work every day, data mismatches, overselling incidents, and a finance team that can't get accurate real-time reporting. Once you're processing 50+ orders a day, this becomes unsustainable.

  • Eliminate manual order entry — save 10–20 hours per week
  • Real-time inventory sync prevents overselling and stockouts
  • Automated refund and return processing
  • Single source of truth for customer data
  • Faster month-end close with accurate, reconciled financials

What Data to Sync

Not everything needs to sync in real time. Here's how I categorise data flows by priority and direction:

Shopify → NetSuite (Real-time, via webhooks)

  • Orders: New orders → NetSuite Sales Orders within seconds of checkout
  • Refunds: Shopify refunds → NetSuite Credit Memos automatically
  • Customers: New/updated customers → NetSuite Customer records
  • Fulfilment: Shopify fulfilment updates → NetSuite item fulfilment status

NetSuite → Shopify (Scheduled, every 15 min)

  • Inventory levels: NetSuite stock quantities → Shopify product inventory
  • Product updates: Price changes, new items → Shopify products
  • Tracking info: Fulfilment tracking numbers → Shopify orders

How to Build It: Step-by-Step

Here's a simplified version of the RESTlet that receives Shopify order webhooks and creates Sales Orders in NetSuite:

SuiteScript 2.1 · RESTlet
/**
 * @NApiVersion 2.1
 * @NScriptType Restlet
 * Receives Shopify order webhooks → creates NS Sales Orders
 */
define(['N/record', 'N/search', 'N/log'],
  (record, search, log) => {

  // POST handler - receives Shopify order payload
  const post = (body) => {
    try {
      const order = JSON.parse(body);

      // Check if order already exists (idempotency)
      const existing = findExistingOrder(order.id);
      if (existing) {
        return { status: 'skipped', reason: 'duplicate' };
      }

      // Create Sales Order in NetSuite
      const soRecord = record.create({
        type: record.Type.SALES_ORDER,
        isDynamic: true
      });

      soRecord.setValue({ fieldId: 'entity',
        value: getOrCreateCustomer(order.customer) });
      soRecord.setValue({ fieldId: 'custbody_shopify_id',
        value: order.id });
      soRecord.setValue({ fieldId: 'trandate',
        value: new Date(order.created_at) });

      // Add line items
      order.line_items.forEach(line => {
        soRecord.selectNewLine({ sublistId: 'item' });
        soRecord.setCurrentSublistValue({
          sublistId: 'item', fieldId: 'item',
          value: findNSItem(line.sku)
        });
        soRecord.setCurrentSublistValue({
          sublistId: 'item', fieldId: 'quantity',
          value: line.quantity
        });
        soRecord.commitLine({ sublistId: 'item' });
      });

      const soId = soRecord.save();
      log.audit('Order Created', `SO ${soId} ← Shopify ${order.id}`);

      return { status: 'success', netsuite_id: soId };

    } catch (e) {
      log.error('Order Sync Error', e.message);
      return { status: 'error', message: e.message };
    }
  };

  return { post };
});

Common Pitfalls to Avoid

⚠️ Warning

These are the mistakes I see most often in DIY or poorly built integrations. Avoid them from day one.

  • No idempotency check: Webhooks can fire multiple times. Always check if the order already exists before creating it.
  • No error alerting: Silent failures mean orders go missing. Build email alerts for any sync errors from day one.
  • Syncing inventory in real-time: Inventory doesn't need to sync on every single order. Every 15 minutes is enough and won't hammer your governance limits.
  • No reconciliation job: Webhooks can fail due to timeouts. Always build a nightly reconciliation script that catches any gaps.
  • Hardcoded IDs: Never hardcode NetSuite internal IDs. Use saved searches to look up items/customers dynamically.
  • No sandbox testing: Always build and test in NetSuite Sandbox + Shopify Development store before production.

Cost & Timeline

A well-built custom integration typically takes 4–6 weeks to design, build, test, and go live. The cost as a freelancer is significantly lower than using an agency — typically 40–60% less for the same (or better) quality.

For ongoing maintenance, most integrations are very stable once built. Expect minor updates when Shopify or NetSuite release breaking API changes — usually once or twice a year.
